## 第一层：问答缓存

目的：减少调用大模型次数

流程：

1. 前端传入问题 
2. 对问题做标准化 
3. 去空格 全半角统一 大小写统一 同义词归一 
4. 生成问题向量 
5. 去“历史问答表”里做相似检索 
6. 如果命中高相似问题，直接返回缓存答案 
7. 没命中再继续后面的知识库和大模型流程

## 第二层：知识库检索

目的：让回答基于你的资料，而不是大模型瞎答

流程：

1. 用户问题生成向量 
2. 去知识库切片里查 topK 
3. 结合相似度阈值过滤 
4. 把命中的知识片段拼成 context 
5. 把 问题 + context 发给大模型生成答案

## 第三层：大模型生成

目的：最后兜底

流程：

1. 先查缓存 
2. 再查知识库 
3. 如果知识库有结果，就走 RAG 提示词 
4. 如果知识库没有结果，再决定： 
5. 要么直接回答“知识库暂无相关内容” 
6. 要么允许走通用大模型回答

这个开关最好配置化。
很多企业场景里，不希望模型脱离知识库自由发挥。
